# SaaS Telegram Bot

Платформа для продажи доступа в Telegram-группы и каналы.

## Быстрый старт

### 1. Настройка переменных окружения

```bash
cp .env.example .env
```

Заполните `.env`:
```env
BOT_TOKEN=ваш_токен_от_BotFather
SUPERADMIN_IDS=ваш_telegram_id
POSTGRES_PASSWORD=надёжный_пароль
```

Получить токен: [@BotFather](https://t.me/BotFather) → `/newbot`
Узнать свой ID: [@userinfobot](https://t.me/userinfobot)

### 2. Запуск через Docker

```bash
docker-compose up -d
```

Это запустит:
- **bot** — Telegram бот
- **db** — PostgreSQL база данных  
- **redis** — Redis для FSM состояний

### 3. Добавление бота в группу/канал

1. Добавьте бота в группу или канал
2. Назначьте бота **администратором** с правами:
   - Удаление сообщений
   - Приглашение пользователей
3. Отправьте боту команду `/start` в личные сообщения

## Роли пользователей

| Роль | Описание |
|------|----------|
| **Супер-admin** | Управляет всей платформой (задаётся через `SUPERADMIN_IDS`) |
| **Admin** | Управляет своими группами и каналами |

## Команды управления

```bash
# Запуск в разработке
docker-compose up

# Просмотр логов бота
docker-compose logs -f bot

# Создание миграции
docker-compose exec bot alembic revision --autogenerate -m "описание"

# Применение миграций
docker-compose exec bot alembic upgrade head

# Остановка
docker-compose down
```

## Как работает система оплаты

1. Пользователь выбирает сообщество и тариф
2. Бот показывает QR-код и реквизиты администратора
3. Пользователь нажимает **«Я оплатил»** и отправляет фото чека
4. Администратор получает уведомление с чеком
5. Администратор нажимает **«Подтвердить»** или **«Отклонить»**
6. При подтверждении — подписка активируется автоматически

## Контроль доступа в группе

Если пользователь без подписки пишет в группу:
- ✅ Сообщение немедленно удаляется
- 🔔 Бот отправляет уведомление с кнопкой «Купить доступ»
- ⏱️ Уведомление удаляется через 30 секунд
